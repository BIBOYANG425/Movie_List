## 1. Reference semantics

### 1.0 Row shapes + id canon (the three-table contract)

| | `tv_rankings` (`supabase_tv_rankings.sql:6-28`) | `book_rankings` (`supabase_book_rankings.sql:6-29`) |
|---|---|---|
| `tmdb_id` | `tv_{showId}_s{seasonNum}` (text, `UNIQUE(user_id, tmdb_id)` `:27`) | `ol_{workKey}` e.g. `ol_OL27448W` (`:9`, unique `:28`) |
| vertical cols, NOT NULL | `show_tmdb_id integer NOT NULL` (`:10`), `season_number integer NOT NULL` (`:11`) | — (author is nullable `:15`) |
| vertical cols, nullable | `season_title` (`:13`), `creator` (`:18`), `episode_count` (`:24`) | `author`, `page_count`, `isbn`, `ol_work_key`, `ol_ratings_average real` (`:15,21-24`) |
| `watched_with_user_ids` | added by `supabase_watched_with.sql:3` | in-DDL `:25` |
| shared cols | `title` (= show name `:12`), `year`, `poster_url`, `type='tv_season'`, `genres`, `tier` CHECK, `rank_position`, `notes`, `bracket`, `primary_genre` | same minus season/creator, `type='book'` |

- **`title` is the SHOW name**, season identity lives in `season_title`/`season_number` (`AddTVSeasonModal.tsx:331,339`). Book `title`/`author` come from OpenLibrary docs verbatim (`openLibraryService.ts:175-176`, author = `author_name[0] ?? 'Unknown'`).
- Season id mint: `tv_${show.tmdbId}_s${season.seasonNumber}` (`AddTVSeasonModal.tsx:330`). **Season 0 (Specials) is unreachable** — `getTVShowDetails` filters `season_number > 0` (`tmdbService.ts:905`) — which is what makes the whole-show watchlist sentinel (season_number 0-or-NULL, §1.5) collision-free.
- Genres: TV search results map `genre_ids` through `TV_GENRE_MAP` (compound names like "Sci-Fi & Fantasy", `tmdbService.ts:517-534,599-604`); the ceremony normalizes via `normalizeTVGenres` (compound → movie-vocab, ≤3, `:540-561`) at season-select time (`AddTVSeasonModal.tsx:326`). Books map OL `subject[]` through `SUBJECT_TO_GENRE` + partial-match fallback, ≤5, gated on `ALL_BOOK_GENRES` (`openLibraryService.ts:103-130`).
- Book `globalScore` is derived, never stored: `ol_ratings_average` (0–5) ×2 at read time (`RankingAppPage.tsx:197,735`; `openLibraryService.ts:184`). TV `globalScore` is fetched live (`getTVShowGlobalScore`, `tmdbService.ts:973-987`) and also never stored.
- Reads: one 6-way parallel fetch on mount (`RankingAppPage.tsx:306-340`), mapped by `rowToTVRankedItem`/`rowToBookRankedItem` (`:144-164,183-204`). RLS on both tables = own + followed SELECT, own-only writes (`supabase_tv_rankings.sql:35-61`, `supabase_book_rankings.sql:37-63`).

### 1.1 TV add flow (AddTVSeasonModal — the reference ceremony)

Steps: `search → show_detail → tier → notes → compare` (`AddTVSeasonModal.tsx:31`).

- **Search**: `searchTVShows` via the proxy (`search/tv`, `tmdbService.ts:823-886`) with the same zero-result typo-retry loop as movies (`:866-880`), merged with a local fuzzy pass over the suggestion pools (`AddTVSeasonModal.tsx:259-304`), 350 ms debounce, stale-request guard.
- **Show detail / season grid**: `getTVShowDetails` (`tv/{id}`, seasons array minus Specials, `tmdbService.ts:892-933`). Grid disables already-ranked seasons (`AddTVSeasonModal.tsx:692-699`) and offers per-season bookmark (`:730-738` → `handleSaveSeasonForLater :352-372`).
- **Season select → item**: mints the composite id, show-name title, season-year fallback, normalized genres, `creator = creators[0]`, `episodeCount`, live `globalScore` (`:323-349`).
- **Tier → Notes → H2H**: shared `TierPicker`/`NotesStep`/`ComparisonStep`; the `RankingSession` pool is **TV-only** (`currentItems.filter(i => i.type === 'tv_season')`, `:399`); empty tier short-circuits to rank 0 (`:391-395`). NotesStep Skip clears notes + watched-with (`:771` — the C4 D3 semantics, present here too).
- **Suggestions**: `fetchTVSuggestions('suggestions'|'backfill')` via the `suggestions` edge function (`tmdbService.ts:761-786`; server: `supabase/functions/suggestions/index.ts:63-81,465-668` — show-level recs, excludes ranked/bookmarked shows server-side under the caller's JWT). Client keeps a session-local consumed list capped at 200 (`AddTVSeasonModal.tsx:66-77`) and pre-expands owned season ids to show ids for filtering (`:85-105`).
- **Preselect router** (the contract the iOS coordinator must copy — `AddTVSeasonModal.tsx:204-233`):

| preselected item | route |
|---|---|
| `showTmdbId` set, `seasonNumber` falsy (whole-show bookmark; 0 or NULL both work) | `getTVShowDetails` → `show_detail` season grid (`:204-214`) |
| `seasonNumber` set (season bookmark / TV re-rank) | straight to `tier`, notes + watched-with prefilled (`:215-229`), globalScore backfilled async |
| neither (the B1 hole) | ALSO straight to `tier` — with a season-less item (`:215` matches anything non-null) |

- **Completion** → `handleAddTVItem` (`RankingAppPage.tsx:1338-1348`) → `addTVItem` (`:801-854`): optimistic splice, **own-row full upsert** via `persistTVRankings([single row])` (`:819-824`; row shape `:771-791` — `show_tmdb_id ?? 0`, `season_number ?? 0`, `watched_with_user_ids ?? []`), then `set_tier_order('tv', tier, targetOrder)` (`:825-828`). Event `ranking_add` with `{notes?, year?, watched_with_user_ids?}` (`:830-842`); stub `createStub(mediaType:'tv_season')` fire-and-forget (`:845-851`). **Both fire even when the save failed — B4.** No journal sheet (§1.6).

### 1.2 Book add flow (UniversalSearch → RankingFlowModal)

- **Search is OpenLibrary, not TMDB, and not proxied**: `searchBooks` fetches `https://openlibrary.org/search.json?q=…&limit=10&fields=…` directly (`openLibraryService.ts:142-197`) — keyless, no auth, 8 s timeout. The tmdb-proxy covers TMDB only (`rules.ts:37-51`); there is nothing to allowlist for OL. **iOS should call OpenLibrary directly the same way** — and unlike a browser it CAN set the `User-Agent` header OL's rate policy asks for (~3 req/s, `openLibraryService.ts:4` documents it; the web fetch cannot send it). No typo-retry/fuzzy layer exists for books (ledgered deferred, `2026-07-07-ios-parity-ledger.md:70`).
- **Entry points**: UniversalSearch "Rank" (`UniversalSearch.tsx:176-181` → `handleSearchRankBook`, `RankingAppPage.tsx:1506-1526`) and watchlist "Rank" (`rankFromWatchlist :721-741`). There is no book add-modal with search and no book suggestion engine (the modal shows results only while typing).
- **Ceremony**: `RankingFlowModal` = `tier → notes → compare` (`RankingFlowModal.tsx:22`), pool = the whole passed `currentItems` (callers pass `bookItems`, `RankingAppPage.tsx:1796-1806`), notes/watched-with prefilled from the item (`:46-47` — this is what makes book re-rank carry notes). The `preselectedTier` fast-path (`:52-64`) is dead for books (no caller passes it).
- **Completion** → `handleAddBookItem` (`RankingAppPage.tsx:1326-1336`) → `addBookItem` (`:1089-1129`): own-row upsert via `persistBookRankings` (row shape `:1059-1079` — all five OL columns), `set_tier_order('book', …)` (`:1109-1112`), `ranking_add` event (`:1114-1126`, **fires even on failed save — B4**). **No stub** — `movie_stubs.media_type` CHECK is `('movie','tv_season')` (`20260325_movie_stubs.sql:11`; contract pin "books never get stubs", `docs/contracts/shared-payloads.md:7-10`). No journal sheet.

### 1.3 Management ops (post-C4 state — what C4's audit already covers, delta only)

- Same-tier drops/reorders and cross-tier drags for TV/books are all positions-only via the RPC with optimistic state + rollback-on-error (`handleTVDrop :912-1005`, `handleTVDropOnItem :1007-1053`, `handleBookDrop :1209-1287`, `handleBookDropOnItem :1289-1324`). Same-tier drops are event-silent (deliberate, ledger `:519`).
- Deletes: DELETE + `set_tier_order` compaction + `ranking_remove` (`removeTVItem :856-910`, `removeBookItem :1131-1171`). Orphan semantics identical to movies (stub kept, journal kept, events kept, watchlist not restored).

Movie branch right below (`:1710-1721`) shows the corrected shape (rerankState + raw-item lookup + single `ranking_move`).

- **TV bookmark shapes** (`tv_watchlist_items`, `supabase_tv_rankings.sql:65-80`): season rows carry the full season identity; **whole-show rows use `tmdb_id = tv_{showId}` and `season_number` 0-or-NULL** (schema comments NULL `:70`; web writes 0 via `?? 0`, `RankingAppPage.tsx:689`; contract pin `shared-payloads.md:515-524`). Season bookmarks by the season-grid bookmark (`:352-372`). **`episode_count` has no watchlist column** — set on the item, silently dropped at write (D3).
- **Book bookmarks**: full OL metadata columns (`book_watchlist_items`, `supabase_book_rankings.sql:67-83`; write `RankingAppPage.tsx:1179-1192`).
- **Rank-from-bookmark** (`rankFromWatchlist :721-765`): book → `RankingFlowModal` straight to tier; TV → `AddTVSeasonModal` preselect router (§1.1 table): whole-show (seasonNumber 0/NULL falsy) → season selection; season → straight to tier. Movie → AddMediaModal.

## 2. Findings

### Blocking (web fixes required BEFORE iOS copies the flows)

**B1 — UniversalSearch "Rank" on a TV show skips season selection and mints season-less corrupt rows.**
`handleSearchRankTV` (`RankingAppPage.tsx:1493-1505`) builds the preselected item with `id: show.id` (`tv_{showId}`), **no `showTmdbId`, no `seasonNumber`, raw un-normalized genres**. The modal's preselect router keys season-selection on `preselectedItem.showTmdbId` (`AddTVSeasonModal.tsx:204`), so this item falls into the "full season bookmark" branch (`:215`) and goes **straight to tier**. Completion persists a `tv_rankings` row with `tmdb_id='tv_{showId}'`, `show_tmdb_id=0`, `season_number=0`, `creator=null`, compound genres (`persistTVRankings` coalesces `?? 0`, `:774-775`) — violating the id canon (`shared-payloads.md:491`) and the "never trust `show_tmdb_id=0`" pin (`:526-530`). This is the exact corruption class C3's B2/D5 fixed on the *Save* path (`tvWatchlistItemFromShow`; ledger `:45` verified prod clean) — the *Rank* path was never fixed. iOS's `mapTVRow` rejects such rows on sight (`WatchlistModels.swift:276-277`); a C5 iOS reader hitting web-minted rows like this would silently drop them.
*Fix shape:* route by season-identity, not object shape — in `handleSearchRankTV` pass `showTmdbId: show.tmdbId` (and normalized genres), and harden the modal router to treat "no `seasonNumber`" as show-level whenever the id matches `^tv_\d+$` (derive `showTmdbId` from the id). iOS must implement the hardened router.

**B2 — TV/book re-rank is still delete-first: cancel = permanent loss, completion = remove+add spam.**
TierRow `onRerank` for tv/books (`RankingAppPage.tsx:1701-1709`) fires `removeTVItem`/`removeBookItem` (unawaited: row deleted, tier compacted, `ranking_remove` emitted) *before* opening the modal. Abandoning the ceremony leaves the season/book in neither rankings nor watchlist; a completed re-rank emits `ranking_remove`+`ranking_add` instead of the contract's single `ranking_move` (`shared-payloads.md:449`). The movie branch directly below (`:1710-1721`) is the corrected reference (rerankState → upsert-in-place → `ranking_move` → both tiers compacted via `handleAddItem :1358-1380`). Ledgered at C4 as deferred-to-later-cycle (`ledger:64,514`); C5 is that cycle. iOS `insertRanking` already implements the correct semantics for all media (pre-read + `CeremonyEmission`, `RankingRepository.swift:200-310`) — the web must match before the port, or the two platforms emit different event streams for the same gesture.
*Fix shape:* extend `rerankState` to tv/book (per-vertical or a `{item, media}` union); completion path already exists in `addTVItem`/`addBookItem` upserts — add the source-tier compaction + event-type switch mirroring `handleAddItem`.

**B3 — TV re-rank persists the zh-localized SHOW title into `tv_rankings`, events, and stubs.**
TierRow receives `localizedItems` (`:1690`); the tv/book re-rank branches pass `item` verbatim (`:1704,1707-1708`) into the ceremony, whose completion writes `title: item.title` (`persistTVRankings :776`), event `media_title`, and the 23505 stub refresh. TV ids DO localize (`fetchLocalizedTitle` tv branch, `tmdbService.ts:802-808`; swap at `useLocalizedItems.ts:95-98`), so one zh-mode TV re-rank swaps the canonical title for every viewer — the same B4 the movie branch fixed with a raw-item lookup (`:1717`, contract pin `shared-payloads.md:429`). Books are *accidentally* immune (ol_ ids never localize, D5) but the branch is shape-identical and should get the same lookup.
*Fix shape:* one line per branch — resolve from `tvItems`/`bookItems` by id before passing (`const raw = tvItems.find(i => i.id === item.id) ?? item`).

**B4 — TV/book ceremony completion emits `ranking_add` (and the TV stub) even when the save failed.**
`addTVItem` computes `saveSucceeded` (`:824-828`) but `logRankingActivityEvent` (`:830-842`) and `createStub` (`:845-851`) run unconditionally; `addBookItem` likewise for the event (`:1108-1126`). The movie path returns false BEFORE event+stub (`addItem :533-546`). Result: phantom "ranked X to S" feed cards (and orphan tv stubs) for ranks that never landed — while the bookmark correctly stays (B5 gate), so friends see a rank the owner doesn't have. iOS already has the correct posture (insert throws → return false before `StubWriter`/journal, `RankPersistence.swift:97-110`); the web reference must match so the contract can say "event only after confirmed save" for every vertical.
*Fix shape:* early-return false before the event/stub when `saveSucceeded` is false (exactly the movie shape).

**B5 — Deep-link re-rank routes TV/book items into the MOVIE ceremony and cross-writes `user_rankings`.**
The deep-link `MediaDetailModal` resolves `foundItem` across all three collections (`RankingAppPage.tsx:1811`) and its Re-rank button fires for any ranked media (`MediaDetailModal.tsx:401-407`), but the page's deep-link `onRerank` (`:1851-1863`) unconditionally does the MOVIE flow: `setRerankState`/`setPreselectedForRank` + `AddMediaModal`. Completing it runs `addItem` → upsert into **`user_rankings`** with `type='tv_season'|'book'` (`:513-532`) — vertical columns dropped, the real `tv_rankings`/`book_rankings` row left behind, plus a `movie` stub minted for it (`:565-571`). Net: a cross-table duplicate that then pollutes movie tier scores, exclusion nets, and the H2H pool. Books compound it by rendering as pseudo-movies in the modal at all (§1.6).
*Fix shape:* branch the deep-link `onRerank` on `foundItem`'s source collection (tv → `setPreselectedTVItem`+TV modal; book → `setBookItemToRank`+book modal — note both then inherit B2 until it's fixed); or hide Re-rank for non-movie items until routed. iOS must route re-rank per media from day one.

### Deferred
